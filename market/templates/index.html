Simulation

At this page, simulations can be run, setting the parameters for the simulation yourself. Below, all the possible parameters are explained. NOTE: for all parameters goes: do not make them too large, or else you are in for a long wait.

  • Market Situation

    • Negotation: In this scenario, buyers and sellers negotiate with each other to determine a transaction price.
    • Grocery Store: In this scenario, a seller sets a price for his/her product in advance, and the buyer can either take it or leave it.

  • Days for simulation

    Number of days to run the simulation.

  • Number of buyers

    This is the number of buyers that try to buy products every day. Default value is 100. Maximum number of buyers should lie around 1000.

  • Number of sellers

    The number of sellers that try to sell products every day. Default value is 100. Maximum number of buyers should lie around 1000.

  • Minimum price

    The minimum price possible in the simulation. Agents will not consider prices lower than this value.

  • Maximum price

    The maximum price possible in the simulation. Agents will not consider prices higher than this value.

  • Buyer interactions

    Number of pairs of buyers that will interact at the end of every day. Pairs are selected randomly.

  • Seller interactions

    Number of pairs of sellers that will interact at the end of every day. Pairs are selected randomly.
